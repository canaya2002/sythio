# Sythio URL Strategy & Page Map

## URL Structure Principles

1. Clean, descriptive, keyword-containing slugs
2. Logical hierarchy that mirrors user mental model
3. No unnecessary nesting depth
4. Each URL targets a distinct search intent
5. No two pages compete for the same primary keyword

## Internal Linking Strategy

### Every page should link to:
- At least 2 other content pages (contextual, in-body)
- The pricing page (via CTA)
- A relevant feature or use case page

### Hub-and-spoke model:
- `/features` links to all individual feature pages
- `/use-cases` links to all individual use case pages
- `/compare` links to all comparison pages
- Individual pages link back to hub + cross-link to related pages

### Cross-linking opportunities:
- Feature pages → related use case pages
- Use case pages → relevant feature pages
- Comparison pages → features + pricing
- Audience pages → use cases + features
- Outcome pages → features + product
- All pages → FAQ for common questions
